The Problem

Why Reinforcement Learning for Finance?

Traditional Approach

Static models -- Black-Scholes assumes constant volatility
Requires future knowledge -- Markowitz needs expected returns
Breaks during crises -- 2008, COVID, regime changes
Ignores transaction costs -- fees are an afterthought
vs

RL Approach

Learns from experience -- adapts to any market regime
Optimizes what matters -- risk-adjusted returns directly
Handles non-stationarity -- continuous adaptation
End-to-end -- costs are part of the objective

Under the Hood

How DQN Training Works

1. Observe -- agent reads 16 market features
2. Act -- epsilon-greedy: random early, greedy later
3. Store -- transition in replay buffer (20K capacity)
4. Sample -- random mini-batch breaks correlation
5. Learn -- minimize TD error via backpropagation
6. Stabilize -- target network via Polyak averaging

Key Innovations

Double DQN: Prevents overestimating returns

Dueling: Separate state value from action advantage

Replay Buffer: Reuse rare events (crashes)

# TD Error
target = r + gamma * Q_target(s', argmax Q_online(s',a'))
loss = Huber(Q_online(s,a) - target)

Advanced

LLM Integration: 4 Roles

⚖

Reward Model

LLM evaluates trades: "Was buying here good given overbought RSI?" Score: -1 to +1.

🤖

Policy Agent

LLM reads market state as text, directly decides buy/sell/hold. Zero-shot trading.

📰

Sentiment Encoder

Scores headlines for sentiment/risk/confidence. Augments state: 16-dim to 19-dim.

🌐

World Model

Generates market scenarios for Monte Carlo planning. Agent picks best action across scenarios.
